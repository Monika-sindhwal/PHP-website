Introduction to Forms

Forms Overview

In PHP, forms are essential tools for interacting with users and collecting data. They allow users to input information, which can then be processed by a PHP script.
To retrieve form data, PHP provides superglobals like $_GET and $_POST.

The form submission method can be either GET or POST. To access data from a GET request, use $_GET. For data from a POST request, use $_POST.
The HTML <form> tag is used to create forms. The syntax for using this tag is:

Syntax:

                    <form action="script_url" method="GET/POST">
                         form elements
                    </form>

Form Attributes

In HTML, the <form> tag has various attributes to define how form data is handled. Two essential attributes are action and method.
Action: Specifies the URL where form data should be sent after submission, often pointing to a PHP script for processing.
Method: Specifies how form data is sent. The main options are GET and POST.

  • GET: Data is sent in the URL as parameters, visible in the address bar. Best for non-sensitive data, such as search queries.

    • Example:

                            <form action="search.php" method="get">
                            <label for="query">Search:</label>
                            <input type="text" id="query" name="query">
                            <input type="submit" value="Search">
                        </form>
  • POST: Data is sent in the HTTP request body, not visible in the address bar. Ideal for sensitive data like passwords.

    • Example:

                            <form action="submit.php" method="post">
                            <label for="query">Search:</label>
                            <input type="text" id="query" name="query">
                            <input type="submit" value="Search">
                        </form>

Superglobal Variables in PHP

PHP provides several superglobal variables to handle data in different contexts. Here’s a quick overview:

Superglobal Variable Definition
$_GLOBALS Accesses global variables from anywhere in the script.
$_SERVER Contains server details, headers, paths, and script locations.
$_GET Data from a GET request, used to retrieve form data sent in the URL.
$_POST Data from a POST request, ideal for secure form submissions.
$_FILES Details of uploaded files, such as name, size, and error status.
$_COOKIE Contains cookie data sent by the client’s browser.
$_SESSION Stores session variables, persisting data across multiple pages.
$_REQUEST Combines data from $_GET, $_POST, and $_COOKIE.
$_ENV Environment variables, often containing server configuration data.
